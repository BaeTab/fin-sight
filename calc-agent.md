# 역할(Role)
당신은 시니어 프론트엔드 개발자입니다. 사용자에게 최고의 UX를 제공하는 금융 계산기 웹 애플리케이션을 구축해야 합니다.

# 프로젝트 개요
React와 Vite를 사용하여 '적금/대출 이자 계산기'를 만듭니다. 서버나 DB 없이 클라이언트 사이드에서 모든 계산이 즉시 이루어지는 SPA(Single Page Application)입니다. 사용자가 숫자를 입력하면 결과와 차트가 실시간으로 반응해야 합니다.

# 기술 스택 (Tech Stack)
- Core: React, Vite (Javascript 또는 Typescript 중 자유롭게 선택)
- Style: Tailwind CSS (모던하고 깔끔한 카드 UI 디자인 적용)
- Chart: Recharts (또는 Chart.js) 라이브러리를 사용하여 그래프 구현
- Icons: Lucide-react 또는 React-icons

# 핵심 기능 요구사항 (Functional Requirements)

1. 탭(Tab) 구성
   - 메인 화면 상단에 [예적금 계산기] / [대출 계산기] 탭을 두어 모드를 전환합니다.

2. 예적금 계산기 기능
   - 입력: 월 적립액(또는 예치금), 기간(개월), 연 이자율(%), 이자 과세 여부(일반/비과세/세금우대), 이자 방식(단리/복리)
   - 출력: 원금 합계, 세전 이자, 세후 이자, 최종 수령액
   - 시각화: 시간이 지남에 따라 원금과 이자가 어떻게 쌓이는지 보여주는 '누적 막대 그래프' 또는 '영역 차트'

3. 대출 계산기 기능 (고도화 포인트)
   - 입력: 대출 금액, 대출 기간, 연 이자율(%), 거치 기간
   - 상환 방식 선택 (중요):
     A. 원리금 균등 상환 (매달 내는 돈이 동일)
     B. 원금 균등 상환 (매달 원금이 동일, 이자는 줄어듦)
     C. 만기 일시 상환 (이자만 내다가 마지막에 원금 상환)
   - 출력: 총 대출 이자, 총 상환 금액, 1회차 월 상환금
   - 시각화: 월별 납부금에서 [원금 vs 이자] 비율이 어떻게 변하는지 보여주는 차트
   - 테이블: 월별 상환 스케줄표 (회차, 납입금, 납입원금, 이자, 잔금)

# UI/UX 디자인 가이드
- 레이아웃: 좌측에는 입력 폼(Input Form), 우측(모바일은 하단)에는 결과 요약 카드와 차트가 배치되는 반응형 레이아웃.
- 인터랙션: '계산하기' 버튼 없이, 입력값이 변할 때마다 즉시 결과가 갱신되는 Real-time Calculation을 구현하세요. (단, 성능을 위해 useEffect 사용 시 주의)
- 숫자 포맷: 모든 금액은 3자리마다 콤마(,)를 찍어서 가독성을 높여주세요 (예: 1,000,000원).

# 코드 작성 규칙
- 컴포넌트를 기능별로 분리하세요 (예: `CalculatorForm`, `ResultCard`, `InterestChart`, `AmortizationTable`).
- 복잡한 이자 계산 로직은 별도의 `utils/calculate.js` 파일로 분리하여 순수 함수로 작성하세요.
- 코드는 깔끔하고 주석이 잘 달려 있어야 하며, 에러 처리(0으로 나누기 방지 등)가 포함되어야 합니다.
